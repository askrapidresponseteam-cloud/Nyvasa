# Nyvasa

A curated edition of luxury Indian homes. Vendor portal with AI-assisted onboarding — photo enhancement, editorial framing, and listing copy generated by reading the photographs.

Built on Next.js 15 (app router) + Firebase + Cloudinary + Anthropic.

---

## What is in p0

- Editorial landing page (`/`)
- Four-step listing creation wizard (`/list`)
  - Step 1 — Photo upload with Cloudinary auto-enhancement (before/after view)
  - Step 2 — Particulars (BHK, area, locality, price, features)
  - Step 3 — AI-drafted copy in three variants (editorial / warm / investor), generated by Claude reading the photographs
  - Step 4 — Editorial-style preview, ready for submission
- Cloudinary signed-upload API route
- Anthropic-powered copy generation API route
- Firebase client + admin SDK plumbing (auth, firestore, storage)
- Draft persistence in localStorage so the vendor can resume

## What is NOT in p0 (planned)

- Phone-OTP auth
- Submission to Firestore (the Submit button is wired but does not write yet — flagged in `ReviewStep.tsx`)
- Verification flows (title doc, ID, video walkthrough)
- Admin moderation queue
- Public browse / search
- IP-based city detection
- Quality scoring with thresholds + nudges
- Editorial framing PNG export

## Setup

### 1. Install

```bash
cp .env.example .env.local
npm install
```

### 2. Firebase project

Create a new project at `console.firebase.google.com` (DO NOT reuse the RRT project). Enable:
- Authentication (Phone provider for India OTP)
- Firestore (start in test mode for now)
- Storage

Copy the web app config into `.env.local` (the `NEXT_PUBLIC_*` keys).

For the admin SDK, generate a service account key (Project Settings → Service accounts → Generate new private key). Copy `project_id`, `client_email`, `private_key` into the non-public keys. Wrap the private key in quotes and keep the `\n` characters intact.

### 3. Cloudinary

Sign up at `cloudinary.com`. The free tier covers early use comfortably. From the dashboard:
- Copy `Cloud name`, `API key`, `API secret` into `.env.local`.

Cloudinary transforms used:
- `e_improve` — auto exposure + contrast
- `e_auto_color` — white balance correction
- `e_sharpen:80` — gentle sharpen
- `q_auto:best, f_auto` — adaptive quality + format per browser
- `c_pad,b_rgb:FAF7F0,w_1.04,h_1.10` — Nyvasa editorial frame (bone padding, wider bottom)

### 4. Anthropic

Get an API key at `console.anthropic.com`. Drop into `.env.local`. The copy generator uses `claude-sonnet-4-6` with multimodal photo input.

### 5. Run

```bash
npm run dev
```

Open `http://localhost:3000`. Hit "List your home" to walk the wizard.

## Deploying to Vercel

Push to GitHub, import in Vercel. Add the same env vars in the Vercel dashboard. `nyvasa.com` is already pointed at the project — update `index.html` is no longer needed once this Next.js app is the root.

Replace the existing `Nyvasa 1st cut.html` file in the repo with this whole project structure. Vercel will detect Next.js automatically and build it.

## Design tokens

| Token | Hex | Use |
|---|---|---|
| `bone` | `#FAF7F0` | Background |
| `ink` | `#1A1612` | Primary text |
| `loden` | `#26352B` | Deep accent (CTA backgrounds, brand) |
| `sienna` | `#B85C3A` | Editorial highlight (only on type, never fill) |
| `smoke` | `#6B6259` | Meta text |
| `hairline` | `#D9D2C5` | Borders |

Type: **Fraunces** (display + body, variable with optical sizing) + **Inter** (UI labels, meta).

Signature element: every listing carries an `№` edition number, e.g. `№ 142 / Bandra West`.

## Next-up after p0

In rough priority order:

1. **Wire submit** — `ReviewStep` should write the listing to Firestore with status `pending_review`, including all photo Cloudinary IDs and the final copy.
2. **Phone-OTP auth** — gate the wizard behind a 30-second phone sign-in.
3. **Quality scoring** — call Gemini Vision or Claude in parallel during upload, score each photo, surface "we can enhance this, but a reshoot would help" for low scorers, with sample shot guides.
4. **Verification flow** — separate route, document upload, optional in-person scheduling.
5. **Admin moderation** — `/admin` with a queue of pending listings, approve/reject with notes back to vendor.
6. **Public browse** — `/edition` showing live listings as an editorial spread.
